# 多智能体协作平台 — 重构计划书

> 基于 v3.5 React 版本，从"多 Agent 系统"转向"团队协作空间 + AI 助手"
> 日期：2026-06-27 | 作者：Uruzill

---

## 一、核心定位转变

```
旧定位                      新定位
─────────                  ─────────
多 Agent 系统               团队协作空间 + AI 助手
Agent 是主角                Agent 是辅助工具
单人使用                     多人实时协作
固定的 7 步流水线             可配置的工作流 + @agent 按需调用
```

---

## 三、详细页面设计

### 3.1 登录页

| 元素 | 说明 |
|------|------|
| 背景 | 简洁美观，品牌色为主 |
| 产品介绍 | 一句话说明 + 3-4 个核心特色 |
| 登录表单 | 用户名 + 密码 |
| 游客入口 | "无需注册，立即试用" |
| 注册链接 | 引导新用户创建账号 |

### 3.2 主界面

四个大卡片/图标入口，清晰直观，无侧栏。

### 3.3 个人模式 — 工作流选择器

| 区域 | 内容 |
|------|------|
| 预设套件 | 卡片展示：自动、快速、协作、编程优化、写作优化等 |
| 固定套件设计 | 原生 8 Agent，Coder+Tester 绑定，减少配置成本 |
| 自定义工作流 | 用户自己编排的工作流 |
| 模板市场 | 社区分享的工作流，一键导入 |
| 工作流配置 | Agent 开关（每个 Agent 可启用/禁用）|

#### 预设套件模板

```
套件            | Planner | Retriever | Coder | Executor | Tester | Writer | Summarizer | Bot
全自动          | ✅      | ✅        | ✅    | ✅       | ✅     | ✅     | ✅         | ✅
编程优化        | ✅      | ❌        | ✅    | ✅       | ✅     | ❌     | ❌         | ❌
写作优化        | ✅      | ✅        | ❌    | ❌       | ✅     | ✅     | ✅         | ❌
快速问答        | ❌      | ❌        | ❌    | ❌       | ❌     | ❌     | ❌         | ✅
自定义          | 用户自由勾选
```

#### 右侧栏功能

| 区域 | 说明 |
|------|------|
| Agent 开关 | 列出所有 Agent，开关控制启用/禁用 |

| 模型预览 | 当前使用的模型 + 点击跳转到模型管理 |

### 3.6 团队模式 — 组织管理

|------|------|

#### @agent 命令

| 命令 | 功能 | 实现难度 |
|------|------|---------|
| `@agent 总结一下` | 总结最近聊天记录 | 🟢 半天 |
| `@agent 创建待办: xxx` | 解析内容加到待办栏 | 🟢 半天 |
| `@agent 搜索知识库 xxx` | 按文件名搜索文件 | 🟢 已有 API |
| `@agent 解释代码` | 分析粘贴的代码 | 🟡 1 天 |

#### 拖入文件逻辑

```
用户在聊天区拖入一个文件
  ↓
文件自动上传到共享知识库
  ↓
聊天框里显示一条系统消息:
  "📎 用户A 上传了 架构设计.docx"
```

共享知识库**不需要向量切片**——只需要文件列表 + 文件名搜索 + 下载。Agent 处理文档时做一次性全文读取即可。

### 3.8 知识库管理（独立页面）

| 区域 | 内容 |
|------|------|
| 文件列表 | 文件名 / 大小 / 上传者 / 日期 / 操作 |
| 上传区 | 拖拽 + 点击上传 |
| 搜索 | 按文件名搜索 |
| 索引状态 | 已索引文件数 / 切片数（仅个人模式） |

**个人模式**：文件进行切片 + 向量化，Agent 语义检索
**团队模式**：文件只存储 + 文件名搜索，无需切片

### 3.9 模型管理（设置页）

| 区域 | 内容 |
|------|------|
| 系统默认模型 | 只读显示当前默认模型 |
| 自定义 API Key | 输入 Key，覆盖系统默认 |
| 自定义模型 | 添加名称 / 接口 URL / API Key |
| 角色模型映射 | 每个 Agent 角色选择不同模型 |

## 四、阶段规划

### 第一阶段：快速见效（1-2 周）

**目标**：让现有系统结构合理，去掉多余的 workspace/project 层级

| 任务 | 工作量 |
|------|--------|
| 登录页视觉升级 + 产品引导 + 游客模式 | 2 天 |
| 主界面四入口重构（个人/团队/知识库/模型） | 3 天 |
| 个人模式工作流选择器（预设套件） | 2 天 |
| 聊天界面三栏布局（右栏 Agent 开关） | 2 天 |
| 团队模式基础（组织 CRUD + 成员） | 2 天 |
| 知识库独立页面（文件列表 + 上传） | 1 天 |
| **合计** | **约 12 天** |

### 第二阶段：团队协作 + Agent 工具化（2-3 周）

**目标**：团队模式可用，@agent 基础功能

| 任务 | 工作量 |
|------|--------|
| 团队聊天（消息广播 + 频道概念） | 3 天 |
| @agent 总结 / 创建待办 / 搜索知识库 | 2 天 |
| 待办栏 + 完成勾选 | 1 天 |
| 拖入文件自动存知识库 | 1 天 |
| 共享知识库在聊天左侧栏嵌入 | 1 天 |
| 工作流自定义（Agent 开关配置化） | 2 天 |
| 模型管理页面（API Key + 模型映射） | 1 天 |
| **合计** | **约 11 天** |

### 第三阶段：高阶功能（可选，依时间而定）

| 功能 | 工作量 |
|------|--------|
| 工作流编辑器画布拖拽（react-flow） | 2-3 周 |
| 自定义 Agent（提示词编辑器） | 1 周 |
| 模板市场（导入/导出工作流） | 1 周 |
| 调试面板（多提示对比） | 1 周 |
| 站内通知 / Webhook / 邮件提醒 | 可选 |

## 五、技术可行性总结

| 模块 | 可行性 | 关键点 |
|------|--------|--------|
| 登录页 + 游客 | ✅ 纯前端 | sessionStorage + 无 token 降级 |
| 四入口重构 | ✅ 前端路由调整 | 当前 AppShell 拆分为多模式 |
| 工作流选择器 | ✅ 前端 | 卡片组件 + 本地状态管理 |
| Agent 开关 | ✅ 前后端 2 天 | 后端动态构建 LangGraph |
| 团队聊天 | ✅ 后端 SSE 已支持多连接 | 消息存数据库 + 广播 |
| @agent | ✅ 已有 LLM 调用 | 封装成工具函数 |
| 共享知识库 | ✅ 后端文件系统 | 按 workspace_id 隔离目录 |
| 画布拖拽 | ⚠️ 工作量大 | 建议放第三阶段 |
