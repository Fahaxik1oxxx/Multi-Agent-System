## 第一部分：用户模拟与细节发现

### 1.1 首次访问场景

> 用户小李是计算机系学生，听说这个平台有多 Agent 功能，打开网站。

**当前问题**：
- 打开 `localhost:5173` 直接跳转到登录页，冰冷的表单，没任何介绍
- 小李不知道这网站是干嘛的，也不知道值不值得注册
- 没有游客模式，想试试必须先注册
- 注册完进去，看到空的工作空间列表，不知道下一步干什么

**细节结论**：
- 登录页必须让用户在登录前就理解产品价值
- 必须提供游客模式降低试用门槛
- 注册后的新用户需要引导

**详细交互细节**：

#### 左侧栏 — 会话管理

| 元素 | 行为 |
|------|------|
| 会话列表 | 按最后活跃时间倒序，显示标题和消息条数 |
| 当前会话 | 高亮背景，不同底色 |
| 新建对话 | 清空主区显示欢迎语，不离开当前工作流 |
| 搜索会话 | 实时搜索，匹配标题和消息内容 |
| 删除会话 | 悬停出现删除按钮，点击确认后删除 |
| 切换会话 | 立即加载该会话的所有消息（含思维链） |

#### 主聊天区 — 消息流

| 元素 | 行为 |
|------|------|
| 欢迎区域 | 显示当前工作流名称、已启用 Agent 数量 |
| 消息气泡 | 用户蓝色右对齐，助理白色左对齐 |
| 思维面板 | 默认折叠，显示 Agent 链摘要，点击展开每个 Agent 卡片 |
| Agent 卡片 | 显示 Agent 名称、彩色标签、输出内容（Markdown 渲染） |
| 助手工具栏 | 复制 / 重新生成 / 生成报告（有思维链时） |
| 用户消息工具栏 | 复制 / 修改（修改后截断后续对话重新发送） |

#### 右侧栏 — Agent 控制

| 元素 | 行为 |
|------|------|
| 工作流名称 | 显示当前使用的工作流 |
| Agent 开关 | 每个 Agent 一行，开关可点击，灰色=禁用，蓝色=启用 |
| 开关操作 | **修改后即时生效，下一次发送消息时使用新配置** |
| 运行时状态 | Agent 正在执行时显示旋转动画 |
| 知识库预览 | 显示文件数量，点击"管理"跳到知识库页面 |
| 模型预览 | 显示当前模型名称，点击"设置"跳到模型管理 |

**Agent 开关的运行时语义**：

```
用户发送消息时:
  ├─ 读取当前右侧栏的 Agent 开关状态
  ├─ 将启用的 Agent 列表传给后端
  └─ 后端根据列表动态构建工作流

执行过程中:
  └─ Agent 开关置灰（不可修改）
     └─ 执行完毕后恢复可修改

用户中途开关 Agent:
  不影响当前正在执行的消息
  影响下一次发送的消息
```

#### 模式切换

| 模式 | 行为 |
|------|------|
| 自动 | Router 判断走快车道还是慢车道 |
| 快速 | 强制走 Bot 单 Agent |
| 协作 | 强制走完整流水线 |

模式切换对当前工作流的影响：**模式是一个快捷入口，覆盖当前工作流的默认行为。** 比如"编程优化"工作流默认走 Coder+Tester，但切换到"快速"时只走 Bot。

#### 输入框

| 元素 | 行为 |
|------|------|
| 多行文本域 | 自动增高，最大 200px |
| Enter 发送 | Shift+Enter 换行 |
| 模式切换标签 | 点击切换，当前模式高亮 |
| 附件按钮 📎 | 上传文件（支持拖拽），发送时附带文件引用 |
| 发送按钮 ➤ | 发送中变为 ■ 终止按钮 |
| 快捷键 | Ctrl+Enter 发送，Esc 取消编辑 |

| 消息可见性 | 仅自己 | 组织内所有成员 |
| Agent 调用 | 完整工作流 | @agent 命令 |
| 知识库 | 个人文档（切片检索） | 共享文件（文件名搜索） |
| 侧栏 | 会话历史 | 文件列表 + 待办 |
| 频道 | 无 | 多个话题频道 |
| 在线状态 | 无 | 显示成员在线/离开 |

**@agent 命令详情**：

| 命令 | 示例 | 回复位置 |
|------|------|---------|
| `@agent 总结一下` | `@agent 总结一下今天的讨论` | 在聊天中回复 |
| `@agent 创建待办` | `@agent 创建待办: 修复登录bug @用户B` | 添加到右侧待办栏，同时聊天中回复确认 |
| `@agent 搜索` | `@agent 搜索知识库 API 文档` | 返回匹配的文件列表 |
| `@agent 解释` | `@agent 解释一下这段代码` + 粘贴代码 | 在聊天中解释 |
| `@agent --deep` | `@agent --deep 分析项目结构` | 触发完整流水线，输出报告 |

**待办功能**：

| 操作 | 行为 |
|------|------|
| 新建待办 | 手动输入，或 @agent 自动创建 |
| 完成勾选 | 点击复选框，标记为已完成（划线效果） |
| 删除待办 | 悬停出现删除按钮 |
| 指派 | @用户名 关联到具体成员 |
| 通知 | 完成后在聊天频道发系统消息"🟢 待办已完成" |

**文件拖入上传流程**：

```
用户拖拽文件到聊天区域
  ↓
系统提示"释放以上传文件到共享知识库"
  ↓
文件上传到服务器（存储到组织目录）
  ↓
聊天框出现系统消息:
  "📎 用户A 上传了 架构图.png"
  ↓
左侧知识库列表刷新
```
