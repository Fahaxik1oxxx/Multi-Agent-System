### 2.2 Router 意图分类

`router.py` 是独立于 Agent 编排的轻量分类器，使用 urllib 直接调 DeepSeek API：

```
用户输入 → urllib POST /chat/completions
         → temperature=0, max_tokens=50, thinking disabled
         → 解析 "标签|复杂度"
         → 返回 (task_type, complexity)
```

**5 种标签：** 编程 / 写作 / 分析 / 问答 / 闲聊
**2 种复杂度：** 轻（快车道）/ 重（慢车道）

**关键词覆写（Router 后执行）：**

| 优先级 | 检测 | 覆写 |
|--------|------|------|
| 1 | 搜索/查资料/检索/基于知识库 | complexity=重 |
| 2 | 以"分析"开头 | complexity=重, task_type=分析 |
| 3 | C语言/Java/Rust/Go/C++/C# | task_type=问答, complexity=轻 |

### 2.3 Agent 清单（7 个）

| Agent | 图标 | LLM | 工具 | 职责 |
|-------|:----:|-----|------|------|
| **Planner** | 📋 | DeepSeek V4 Flash | 无 | 拆解任务为步骤清单，输出 task_type |
| **Bot** | 🤖 | DeepSeek V4 Flash | 无 | 快车道直接回复，唯一面客角色 |
| **Retriever** | 🔍 | DeepSeek V4 Flash | `search_knowledge` | 查询 ChromaDB 知识库 |
| **Coder** | 💻 | DeepSeek V4 Flash | `write_file`, `read_file`, `calculate` | Python 代码 + 数据分析 |
| **Writer** | ✍️ | DeepSeek V4 Flash | `write_file`, `read_file`, `search_knowledge` | 报告/方案/文章 |
| **Tester** | ✅ | DeepSeek V4 Flash | `read_file` | 代码+文档质量评审 |
| **Summarizer** | 📊 | DeepSeek V4 Flash | 无 | 汇总执行过程 → 结构化报告 |

### 2.4 LangGraph 工作流状态

```python
class WorkflowState(TypedDict):
    user_input: str
    lane_mode: str          # "auto" | "fast" | "slow"
    task_type: str          # "编程" | "写作" | "分析" | "问答" | "闲聊"
    complexity: str         # "轻" | "重"
    plan: str
    knowledge: str
    code_or_draft: str
    execution_result: str   # 含 exitcode 的 stdout/stderr
    test_result: str
    fix_count: int          # 当前修复轮数 (max 2)
    agent_messages: list    # 所有中间消息
    speaking_log: list      # [{from, to}] 发言流转记录
    final_output: str
```

**5 个条件路由点：**

| 路由点 | 条件 | 结果 |
|--------|------|------|
| 入口 | `complexity == "轻"` | → Bot |
| 入口 | `complexity == "重"` | → Planner |
| Retriever 后 | `task_type == "写作"` | → Writer |
| Retriever 后 | `task_type in ("编程","分析")` | → Coder |
| Executor 后 | `exitcode == 0` | → Tester |
| Executor 后 | `exitcode ≠ 0` | → Summarizer（或 Coder 修复） |
| Tester 后 | `✅ in test_result` | → Summarizer |
| Tester 后 | `❌ in test_result and fix_count < 2` | → Coder/Writer |
| Tester 后 | `fix_count >= 2` | → Summarizer（强制结束） |

## 三、关键设计决策

### 3.1 决策总览

| # | 决策 | 选择 | 理由 |
|---|------|------|------|
| 1 | **Streamlit → FastAPI Web** | 全面迁移 | 标准 Web 技术栈，无框架锁定，可扩展 |
| 2 | **Router 恢复** | 独立 LLM 分类器 | 15% 误判被关键词覆写消除，自动化 + 兜底兼备 |
| 3 | **自动/手动双模式** | auto（默认）+ fast + slow | 日常用自动，特殊需求手动覆写 |
| 4 | **exitcode 分支** | executor 输出 exitcode → 路由决策 | 比纯文字 Tester 判断更精确可靠 |
| 5 | **5 任务类型恢复** | 编程/写作/分析/问答/闲聊 | 恢复 v2.x 完整分类，分析类型走专用 Coder |
| 6 | **前端折叠思考** | 默认折叠 + 点击展开 | 减少视觉噪音，用户按需查看细节 |
| 7 | **Jinja2 + Bootstrap 5** | 服务端渲染 + CDN | 零 Node.js 依赖，标准 Python Web 栈 |

### 3.2 决策 1 详解：为什么从 Streamlit 迁移到 FastAPI Web

**Streamlit 的限制（v3.0 实际遇到的）：**

1. **页面刷新模型**：每次状态变化需要 `st.rerun()` 全页刷新，无法实现精细 DOM 操作
2. **组件不可定制**：Streamlit 的 `st.chat_message`、`st.expander` 等组件样式受限
3. **部署耦合**：只能 `streamlit run`，无法融入标准 ASGI 生态（Nginx 反代、Gunicorn worker）
4. **SEO/分享**：纯 SPA 渲染，无法被搜索引擎索引

**FastAPI Web 的改善：**

1. **标准 HTTP**：普通 GET/POST，可用任何 HTTP 工具测试和集成
2. **完全 UI 控制**：HTML/CSS/JS 自由定制，不受框架组件限制
3. **ASGI 生态**：可部署到 Uvicorn、Gunicorn、Daphne 等标准服务器
4. **前后端分离潜力**：当前 Jinja2 渲染，未来可改为 React/Vue 前端 + 同一 API

### 3.3 决策 2 详解：Router 恢复 + 关键词覆写

v3.0 移除了 Router，改为手动选择——消除了 15% 误判，但增加了用户认知负担。

v3.1 恢复 Router，但不再单纯依赖它：

```
Router LLM → 关键词覆写（二次修正）→ 最终车道
```

**关键词覆写是安全网：**

| 场景 | Router | 关键词 | 最终 |
|------|--------|--------|------|
| "查一下知识库里的相关资料" | 问答\|轻 | 搜索检测 → 重 | 慢车道 ✅ |
| "分析这个 CSV" | 分析\|重 | — | 慢车道 ✅ |
| "Java 写一个 Hello World" | 编程\|轻 | 非Python检测 → 问答轻 | 快车道 ✅ |
| "实现 LRU Cache" | 编程\|重 | — | 慢车道 ✅ |

默认自动模式 + 保留手动覆写 = 最佳体验。

### 3.4 决策 3 详解：exitcode 分支 vs 纯文字评审

v3.0 的 executor 执行代码后，将 stdout/stderr 文本传给 Tester 做文字评审。问题在于：Tester 是 LLM，可能误判——代码已经跑通了但 Tester 说"看起来有问题"。

v3.1 加入 exitcode 驱动分支：

- `exitcode: 0` → 代码执行成功 → 进入 Tester 评审
- `exitcode ≠ 0` → 代码执行失败 → 返回 Coder 修复（含 stderr）
- 修复超过 2 次 → 强制进入 Summarizer

这是一个确定性的信号（exitcode 是操作系统返回的），不受 LLM 幻觉影响。

### 5.3 慢车道完整流水线

**输入：** "写一个 Python 函数计算阶乘"  
**车道：** 自动 → Router 判定 编程\|重 → 慢车道

| 步骤 | Agent | 产出 | speaking_log |
|:----:|-------|------|-------------|
| — | 🧠 Router | 编程\|重 | — |
| 1 | 📋 Planner | 5 步执行计划 | 用户→Planner→Retriever |
| 2 | 🔍 Retriever | 知识库检索（如有） | Planner→Retriever→Coder |
| 3 | 💻 Coder | `def factorial(n)` 完整实现 | Retriever→Coder→Executor |
| 4 | ⚙️ Executor | exitcode: 0, stdout: 120 | Coder→Executor→Tester |
| 5 | ✅ Tester | ✅ 评审全部通过 | Executor→Tester→Summarizer |
| 6 | 📊 Summarizer | 结构化 Markdown 报告 | Tester→Summarizer→结束 |

**最终产出：** 代码实现 + 执行结果 + 评审结论 + 完整报告。

### 5.4 exitcode 修复循环

**输入：** "写一个函数，故意缺少 import"（触发 NameError）

| 阶段 | Agent | 结果 |
|------|-------|------|
| 首次 | Coder → Executor | exitcode: 1, NameError: name 'xxx' is not defined |
| 修复 1 | Coder（含上次 stderr） | 补充 import → Executor → exitcode: 0 |
| 评审 | Tester | ✅ 通过 |

## 六、3.1 新增能力

### 6.1 Streamlit → FastAPI Web 全面迁移

| 维度 | v3.0 (Streamlit) | v3.1 (FastAPI Web) |
|------|-----------------|-------------------|
| 页面渲染 | `st.chat_message` 组件 | Jinja2 模板 + JS 动态 DOM |
| 样式控制 | `st.markdown(unsafe_allow_html)` | Bootstrap 5 + 独立 CSS 文件 |
| 路由 | 隐式（Streamlit 管理） | 显式 REST API（7 个端点） |
| 静态文件 | Streamlit 内置 | `/static` + `/coding` 挂载 |
| 部署 | `streamlit run` | `uvicorn main:app`（ASGI 标准） |
| SEO | ❌ 纯 SPA | ✅ 服务端渲染 HTML |

### 6.2 Router 意图分类恢复

- `router.py`：独立于 Agent 编排的轻量 LLM 调用（urllib + temperature=0）
- 5 种任务类型（编程/写作/分析/问答/闲聊）+ 轻/重复杂度
- 关键词覆写三层安全网：搜索检测 → 分析检测 → 非 Python 降级

### 6.3 exitcode 驱动分支

- Executor 输出 `exitcode: 0/1/...` → 工作流条件路由
- exitcode ≠ 0 自动返回 Coder 修复（≤2 次）
- 2 次后强制进入 Summarizer，避免死循环

### 6.4 speaking_log 发言记录

## 七、总结与展望

### 7.1 项目成果

| 指标 | v3.0 | v3.1 |
|------|------|------|
| 源文件 | 15 | **22** |
| Agent 数量 | 7 | 7 + **Router**（分类器） |
| 任务类型 | 2（coding/writing） | **5**（编程/写作/分析/问答/闲聊） |
| 车道模式 | 手动 2 种 | **自动 + 手动 3 种** |
| 编排分支 | Tester 文字判断 | **exitcode + Tester 双保险** |
| 前端文件 | 1（main.py） | **7**（3 HTML + 1 CSS + 1 JS + main.py + knowledge.py） |
| API 端点 | 0（Streamlit 隐式） | **7 个显式 REST 端点** |
| 测试覆盖 | 0 | **6 个自动化测试** |
| 启动命令 | `streamlit run` | `uvicorn main:app` |

### 7.2 关键收获

1. **标准 Web > 框架锁定**：FastAPI + Jinja2 + Bootstrap 是 Python Web 的"通用语言"——任何 Python 开发者都能理解和维护，不像 Streamlit 有学习曲线和框架限制
2. **自动化 + 安全网 = 最优解**：Router 自动分类覆盖 85% 场景，关键词正则兜底剩余 15%——纯手动或纯自动都不如两者结合
3. **确定性信号 > LLM 判断**：exitcode 是操作系统返回的确定性值，比 LLM 的"看起来没问题"可靠得多——能用代码判断的就不要问 LLM
4. **LangGraph 的显式图优势**：加 exitcode 分支只需加一个 `add_conditional_edges`，比 AG2 的 if/else 状态机清晰 10 倍
5. **前端折叠是必要的**：多 Agent 系统产生大量中间输出——默认折叠把主导权还给用户，比全部展开信息轰炸好得多

### 7.3 可改进方向

- **WebSocket 流式推送**：当前一次性返回，改为 SSE/WS 逐 Agent 推送可大幅提升感知速度
- **Docker 沙箱**：subprocess 在宿主机执行，升级为 Docker 容器隔离更安全
- **并行 Agent 执行**：LangGraph 支持 `Send()` API，可将独立任务（如代码编写+文档编写）并行化
- **前端框架升级**：当前原生 JS 可平滑迁移到 Alpine.js 或 htmx，保持零构建工具
- **A/B 模型对比**：同一任务同时跑两个不同模型，对比输出质量
- **持久化记忆**：当前仅注入最近 3 轮摘要，可升级为长期记忆层（如 Mem0 / LangChain Memory）
