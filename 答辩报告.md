## 一、项目概述

### 1.1 项目简介

本项目构建了一个**由 7 个专业角色 Agent 组成的多智能体协作系统**，包含规划 Agent、检索 Agent、执行 Agent、编程 Agent、写作 Agent、评审 Agent 和总结 Agent。用户用自然语言描述需求，系统通过 **Router 意图分类器**自动识别任务类型和复杂度，由 **LangGraph 状态图**编排多个 Agent 按流程协作，最终输出结构化任务报告与完整执行日志。

v3.3 新增**用户认证与隔离体系**：注册用户通过 bcrypt 密码哈希和 UUID Token 进行安全认证，会话和知识库完全按用户隔离，游客模式使用浏览器 sessionStorage 存储对话。

### 1.2 核心能力一览

| 能力维度 | 实现方式 |
|----------|----------|
| **多角色协作** | 7 个专业 Agent，LangGraph StateGraph 统一编排 |
| **意图识别** | Router LLM 自动分类 5 种任务类型 + 轻/重复杂度 + 关键词覆写安全网 |
| **任务拆解** | Planner Agent 自动将复杂任务拆解为可执行步骤清单 |
| **知识检索** | ChromaDB 向量数据库 + BAAI/bge-small-zh-v1.5 嵌入模型，按用户物理隔离 |
| **代码执行** | subprocess 沙箱隔离执行，exitcode 驱动确定性分支路由 |
| **结果校验** | Tester Agent 代码评审 + 文档质量检查 + ≤2 轮自动修复循环 |
| **工具链协同** | 7 个 LangChain Tool：搜索、计算、文件读写、数据分析、可视化、OCR |
| **结构化报告** | Summarizer Agent 汇总全流程，输出 Markdown 报告 + 执行日志 |
| **Web 交互** | FastAPI + Jinja2 + daisyUI + Tailwind CSS 现代 Web 界面 |
| **用户认证** | bcrypt 密码哈希 + UUID Token 鉴权，注册/登录/退出完整流程 |
| **用户隔离** | 会话归属校验（403 拦截跨用户访问），知识库按 user_id 物理分目录 |
| **游客模式** | 无需注册即可聊天，对话存 sessionStorage，关标签页即清除，登录可迁移 |

### 4.3 认证数据流

```
注册：POST /api/auth/register {name, email, password}
  → bcrypt.hashpw(password)
  → INSERT INTO users (id, name, email, password_hash)
  → INSERT INTO auth_tokens (uuid4, user_id)
  → 创建 rag/documents/<user_id>/ 和 rag/chroma_db/<user_id>/
  → 返回 {token, user_id, name}

登录：POST /api/auth/login {name, password}
  → SELECT FROM users WHERE name = ?
  → bcrypt.checkpw(password, hash)
  → INSERT INTO auth_tokens
  → 返回 {token, user_id, name}

鉴权：每个请求 → Authorization: Bearer <token>
  → require_auth() → SELECT user_id FROM auth_tokens WHERE token = ?
  → 有效 → 注入 user_id 到路由处理函数
  → 无效 → 401
```

### 4.6 数据库设计

```sql
-- 用户表（bcrypt 密码哈希）
CREATE TABLE users (
    id         TEXT PRIMARY KEY,
    name       TEXT NOT NULL UNIQUE,
    email      TEXT DEFAULT '',
    password   TEXT NOT NULL DEFAULT '',
    created_at TEXT DEFAULT (datetime('now', 'localtime'))
);

-- 认证令牌表
CREATE TABLE auth_tokens (
    token      TEXT PRIMARY KEY,
    user_id    TEXT NOT NULL,
    created_at TEXT DEFAULT (datetime('now', 'localtime')),

);

-- 会话表（外键约束 + 归属校验）
CREATE TABLE sessions (
    id         TEXT PRIMARY KEY,

    title      TEXT DEFAULT '',
    messages   TEXT DEFAULT '[]',
    updated_at TEXT DEFAULT (datetime('now', 'localtime')),
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## 五、关键设计决策

### 5.1 决策总览

| # | 决策点 | 选择 | 核心理由 |
|---|--------|------|----------|
| 1 | 多 Agent 框架 | **LangGraph** 替代 AG2 | 显式 StateGraph，可调试可扩展 |
| 2 | 前端框架 | **FastAPI Web** 替代 Streamlit | 标准 HTTP，完全 UI 控制 |
| 3 | 意图分类 | **Router LLM + 关键词覆写** | 自动化覆盖 85% + 安全网兜底 15% |
| 4 | 执行判断 | **exitcode 确定性分支** | 不受 LLM 幻觉影响 |
| 5 | CSS 框架 | **daisyUI 5** 替代 Bootstrap | 语义组件更丰富 |
| 6 | 数据持久化 | **SQLite WAL** 替代 localStorage | 跨设备、多用户、并发支持 |
| 7 | 密码安全 | **bcrypt** 替代明文/localStorage | 服务端哈希，不可逆 |
| 8 | 认证机制 | **UUID Token** 替代 JWT | 轻量，无过期管理开销，适合答辩项目 |
| 9 | 知识库隔离 | **物理分目录** 替代元数据标记 | 删用户 = 删目录，零残留 |

### 5.2 决策详解

**决策 7：bcrypt 密码哈希**

v3.2 的"注册"本质是 localStorage 存明文邮箱密码，安全风险高。v3.3 升级为：
- 服务端 bcrypt 哈希存储，密码原文永不留存
- `bcrypt.hashpw(password, bcrypt.gensalt())` 自动生成随机盐
- `bcrypt.checkpw()` 恒时比较，防时序攻击
- 前端不再存密码，仅存 Token

**决策 8：UUID Token vs JWT**

JWT 需要过期刷新、签名密钥管理。本项目是答辩/实习场景，UUID Token 更合适：
- 实现简单：`str(uuid.uuid4())` 即生成，数据库即状态
- 退出即删除：DELETE FROM auth_tokens WHERE token = ?，无需黑名单
- 零配置：不需要 JWT_SECRET 环境变量

**决策 9：知识库物理隔离 vs 元数据标记**

物理方案的优势：
- 每个用户独立的 ChromaDB PersistentClient 和目录
- 检索时天然隔离，不需要 `WHERE user_id = ?` 过滤
- 删除用户：`rm -rf rag/documents/<user_id>/ rag/chroma_db/<user_id>/`

**游客 sessionStorage vs 数据库**

v3.2 将所有用户（含游客）的会话都存入 SQLite。v3.3 改为：
- 游客会话 → sessionStorage（关标签页即清除，不占数据库空间）
- 注册用户会话 → SQLite（持久化、跨设备）
- 登录时自动迁移：sessionStorage → POST /api/sessions

| 测试场景 | 操作 | 结果 |
|----------|------|------|

### 6.4 游客模式实测

| 操作 | 结果 |
|------|------|
| 首次访问（未登录） | 显示"游客"，可正常发送消息 |
| 发送消息后刷新页面 | 对话保留（sessionStorage） |
| 关闭标签页重开 | 对话消失（sessionStorage 清除） |
| 知识库操作 | 返回 401（游客无权访问） |
| 游客发消息 → 注册登录 | 对话迁移到侧栏历史 |

## 七、v3.3 核心亮点

### 7.1 用户认证系统（bcrypt + UUID Token）

从 localStorage 明文密码升级为标准服务端认证：

| 对比维度 | v3.2 | v3.3 |
|----------|------|------|
| 密码存储 | localStorage 明文 | SQLite bcrypt 哈希 |
| 认证方式 | 前端自行比对 | 后端 Token 鉴权 |
| 注册 | 仅存 localStorage | `POST /api/auth/register` |
| 退出 | 清除 localStorage | 删除服务端 Token |
| 安全级别 | 无 | 4 层防护 |

**4 层安全防护：**

| 防护层 | 实现 |
|--------|------|
| 密码哈希 | bcrypt + 随机盐，恒时比较 |
| Token 鉴权 | UUID，每次登录生成新 Token |
| SQL 注入 | 100% 参数化查询 |
| 路径遍历 | `os.path.basename()` 消毒 |

### 7.2 用户数据隔离

| 维度 | v3.2 | v3.3 |
|------|------|------|
| 会话归属 | 无校验（传什么 user_id 就存什么） | Token 解析 + 403 拦截 |
| 知识库 | 全局共享 `rag/documents/` `rag/chroma_db/` | `rag/documents/<user_id>/` `rag/chroma_db/<user_id>/` |
| 知识库 API | 无需登录 | 全部需 Token |

### 7.3 游客模式升级

| 维度 | v3.2 | v3.3 |
|------|------|------|
| 会话存储 | SQLite（作为"游客_xxx"数据库用户） | sessionStorage（纯浏览器端） |
| 后端痕迹 | 创建数据库用户记录 | 零后端存储 |
| 刷新行为 | 从数据库恢复 | 从 sessionStorage 恢复 |
| 关闭标签页 | 保留（数据库） | 清除（sessionStorage） |
| 登录迁移 | 无 | 自动迁移到用户数据库 |

### 7.4 开发体验改善

| 改善项 | 说明 |
|--------|------|
| `.env` 自动加载 | 项目启动时 `load_dotenv()` 自动加载 `.env`，不再依赖系统环境变量 |
| 错误信息优化 | 500 错误显示具体原因（如 "DEEPSEEK_API_KEY 未设置"），不再模糊 |
| 数据库容错 | `save_session` 外键容错——数据库重建后自动补建用户记录 |
| 端口冲突 | 从 8501 改为 8502，避开被占用的端口 |

## 九、总结与展望

### 9.1 项目成果汇总

| 指标 | v3.1 | v3.2 | v3.3 |
|------|:----:|:----:|:----:|
| 源文件数 | 22 | 24 | **26** |
| Agent 数量 | 7+Router | 7+Router | 7+Router |
| API 端点 | 7 | 18 | **22** |
| 测试用例 | 6 | 6 | **9** |
| 数据持久化 | localStorage | SQLite | SQLite + Token |
| 用户系统 | 无 | 幂等注册 | **bcrypt + Token + 隔离** |
| 知识库 | 全局共享 | 全局共享 | **按用户物理隔离** |
| 游客模式 | 数据库存储 | 数据库存储 | **sessionStorage** |
| 安全防护 | 基础 | 4 层 | **7 层** |

### 9.2 关键收获

1. **LangGraph 显式图 > 黑盒调度**：工作流即文档，加分支一行代码
2. **确定性信号 > LLM 判断**：exitcode 比 LLM 文字判断可靠得多
3. **标准 Web > 框架锁定**：FastAPI + Jinja2 + daisyUI，任何 Python 开发者都能维护
4. **持久化是产品分水岭**：SQLite + WAL 让系统从 Demo 升级为可用产品
5. **认证是数据隔离的前提**：有了可靠的用户身份，会话和知识库隔离才有意义
6. **物理隔离 > 逻辑过滤**：分目录比加 WHERE 子句更简单、更安全、更直观
7. **游客也是用户**：sessionStorage 让游客体验流畅，登录后迁移让数据不丢失

### 9.3 后续可改进方向

| 方向 | 描述 | 优先级 |
|------|------|:------:|
| **WebSocket 流式推送** | 逐 Agent 推送进展，提升感知速度 | 高 |
| **Docker 沙箱隔离** | subprocess 升级为容器隔离 | 高 |
| **并行 Agent 执行** | LangGraph `Send()` API 并行化独立任务 | 中 |
| **会话密码修改** | 添加密码修改和找回功能 | 中 |
| **JWT 升级** | 生产环境可用 JWT 替代 UUID Token | 低 |
| **长期记忆层** | Mem0 或 LangChain Memory | 低 |
